### Что нужно сделать?
- **Описание задачи:** Агентная платформа, позволяющая регистрировать автономных A2A-агентов, подключать LLM-провайдеры через OpenRouter, маршрутизировать запросы и собирать телеметрию. Все три уровня сложности.
- **Job Story:** Когда преподаватель проверяет ДЗ, я хочу показать работающую платформу с реальными агентами, умной маршрутизацией и нагрузочными тестами, чтобы получить максимальный балл (25).

### Полезное действие (Зачем?)
- **Базовая проблема:** Домашнее задание по бонус-треку LLM, инфраструктурный трек ИТМО
- **Ценность:** Максимальный балл (25) за уровень 3. Побочная ценность - реальная инфраструктура для мониторинга LLM-расходов EventAI.

### Жёсткие ограничения (Constraints)
- **Бюджет:** $19 на OpenRouter (бесплатные и дешёвые модели для нагрузки)
- **Сроки:** Реализация за 1 день, общий дедлайн 12.04.2026
- **Команда:** Один разработчик (Дмитрий) + Claude Code
- **Инфраструктура:** Docker Compose, OpenRouter API
- **Регуляторные:** Нет

### Что нужно учесть?
- **LLM-провайдеры:** Через OpenRouter подключаются разные модели как "разные провайдеры" для балансировки
- **Модели для нагрузки:** StepFun Step 3.5 Flash (free), NVIDIA Nemotron 3 Super (free), DeepSeek V3.2 ($0.26/M), gpt-oss-120b ($0.039/M), Grok 4.1 Fast ($0.20/M)
- **Агенты:** Реальные агенты из проекта EventAI (DemoDay) - Profile Agent и Curator Agent с tool use
- **Формат:** Публичный GitHub-репозиторий dpGorbunov/llm-agent-platform

### Критерии приёмки

**Уровень 1 (10 баллов):**
- [ ] Docker Compose поднимает все компоненты
- [ ] Минимум 3 модели через OpenRouter как "провайдеры"
- [ ] Балансировщик: роутинг по моделям, round-robin для реплик, статические веса
- [ ] Streaming ответов без разрыва соединения
- [ ] OpenTelemetry: логи запросов, время ответа
- [ ] Prometheus + Grafana: латентность (p50/p95), трафик по провайдерам, коды ответов, CPU
- [ ] Health-check endpoint для каждого сервиса

**Уровень 2 (20 баллов):**
- [ ] A2A Agent Registry: регистрация агентов с Agent Card, CRUD
- [ ] Динамическая регистрация LLM-провайдеров (URL, цена/токен, лимиты, приоритет)
- [ ] Latency-based routing: приоритет быстрому провайдеру
- [ ] Health-aware routing: исключение провайдера при 5xx/429/timeout
- [ ] Метрики TTFT, TPOT, токены in/out, стоимость запроса
- [ ] MLFlow трассировка агентов и LLM

**Уровень 3 (25 баллов):**
- [ ] Guardrails: фильтр prompt-injection, утечка секретов
- [ ] Авторизация агентов и LLM по токенам
- [ ] Нагрузочные тесты на реальном API (throughput, латентность, отказоустойчивость)

### Out of Scope
- Веб-интерфейс (только API + Grafana)
- Персистентность данных между перезапусками (in-memory)
- Продакшн-деплой на сервер
- Полная интеграция с DemoDay кодовой базой (только адаптеры агентов)
